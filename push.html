GitのPushの使い方

Pushとは

  • コミットした内容を全員で共有するコマンド
  • ローカルリポジトリの内容をリモートリポジトリに反映させる

このpushというコマンドは自分だけのリポジトリに記録されている変更履歴を全員で共有しているリポジトリに反映させることによってみんながpullというアクションをすることによって自分の変更履歴をそれぞれのリポジトリに反映させることができる。それぞれの開発者がコミットをして切りがいいところでpushを行うことによって開発している対象物がリモートリポジトリの中で完成していくわけである。

端末上でのpushのやり方

git push　リモートリポジトリ名 自分のローカルリポジトリ名

とすればpushができる

git push リモートリポジトリ名 自分のローカルリポジトリ名　--force

とすれば強制的なpushをすることができる

SourceTree上でのpushのやり方

SourceTreeとは

  • gitの操作をGUI上でおこうなうことを可能にするソフト

  • CUI上でのコマンド操作ではないので変な操作をしてしまわない
